Glossary / Puzzle Pieces

  • storeAn object {} that holds the state of the application and provides us with methods for manipulating it, including dispatch() and getState().
  • stateThe data that must be remembered in order for the app to function, an object {}.
  • actionAn object {} that represents a change request, similar to an HTTP request. It has a type, which states what to do, and optionally, a payload containing data required to complete the request.
  • dispatchA method available on the store, which accepts an action as an argument. It calls the reducer with the current state in the store and the action in order to reassign the state.
  • reducerA pure function that takes the current state and an action, and then uses (or reduces) that data to return the new state. It contains a switch statement for routing/handling actions.
